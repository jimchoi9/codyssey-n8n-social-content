# 플랫폼별 프롬프트 템플릿 (초안 → 수정 → 최종)

> 제출물 #3. 변수 `{주제}` / `{톤}` 은 워크플로우에서 폼 입력값으로 자동 치환됩니다.
> 실제 워크플로우에 적용된 것은 각 항목의 **최종본**입니다.

---

## 1. 인스타그램

**초안**
```
{주제}에 대한 인스타 캡션 써줘. 해시태그도.
```
→ 문제점: 길이·톤·해시태그 개수 통제가 안 됨. 설명문이 섞여 나옴.

**수정**
```
{주제}로 인스타그램 캡션을 작성해줘. 짧게 쓰고 해시태그 10개 정도 붙여줘.
```
→ 개선: 길이/해시태그 수 명시. 여전히 "다른 설명"이 붙고 줄바꿈이 없음.

**최종 (적용본)**
```
당신은 소셜미디어 카피라이터입니다. 아래 주제로 인스타그램 게시물을 작성하세요.
조건: (1) 캡션은 2~4문장의 짧고 감각적인 한국어 (2) 첫 문장으로 시선을 끌 것
(3) 줄바꿈으로 가독성 확보 (4) 마지막 줄에 관련 해시태그 10개 내외를 한 줄로.
톤앤매너: {톤}. 주제: {주제}.
캡션과 해시태그만 출력하고 다른 설명은 쓰지 마세요.
```

---

## 2. 블로그

**초안**
```
{주제}로 블로그 글 써줘.
```
→ 문제점: 길이·구조 미지정 → 너무 짧거나 소제목 없는 문단 덩어리.

**수정**
```
{주제}로 블로그 글을 500자 이상 써줘. 소제목도 넣어줘.
```
→ 개선: 분량·소제목 지정. 제목 형식과 섹션 밀도가 들쭉날쭉.

**최종 (적용본)**
```
당신은 전문 블로거입니다. 아래 주제로 한국어 블로그 글을 작성하세요.
조건: (1) 매력적인 제목을 첫 줄에 (2) 본문 500자 이상 (3) ##로 시작하는 소제목 2~3개로 구조화
(4) 각 섹션은 2~4문장. 톤앤매너: {톤}. 주제: {주제}. 마크다운으로 출력하세요.
```

## 3. X (트위터)

**초안**
```
{주제}를 트위터용으로 짧게 요약해줘.
```
→ 문제점: 280자 제한을 자주 초과. 해시태그 누락.

**수정**
```
{주제}를 280자 이내로 요약해줘. 해시태그도 넣어줘.
```
→ 개선: 글자수·해시태그 명시. 가끔 설명 문구가 앞에 붙음.

**최종 (적용본)**
```
당신은 카피라이터입니다. 아래 주제를 X(트위터)용으로 요약하세요.
조건: (1) 반드시 280자 이내(한국어 기준) (2) 핵심만 임팩트 있게 (3) 마지막에 해시태그 1~2개.
톤앤매너: {톤}. 주제: {주제}. 트윗 본문만 출력하세요.
```

## 4. 대표 이미지 (Pollinations.ai · 무료)

> `결과 조립` 코드 노드에서 `{주제}`를 넣어 URL을 조립합니다. (별도 키 불필요)

**초안**
```
{주제} 이미지 만들어줘
```
→ 문제점: 비율·스타일·텍스트 통제 안 됨. 소셜용으로 부적합한 결과.

**수정**
```
clean modern vibrant social media hero image about {주제}, minimal text, professional, high quality
```
→ 개선: 용도/스타일 지정. 다만 `minimal text`(최소화)라 이미지에 글자가 종종 들어가고, 한글은 깨져 보임.

**최종 (적용본, 영문이 결과 안정적)**
```
clean modern vibrant social media hero image about {주제}, no text, no words, no letters, no typography, no captions, professional, high quality
```
→ 변경 이유: `minimal text` → **`no text, no words, no letters, no typography, no captions`** 로 강화해 이미지 내 글자(특히 깨지는 한글)를 최대한 배제.
→ URL 옵션: `width=1024&height=1024`(1:1) · `nologo=true`(워터마크 제거) · `seed=42`(고정 시드).

## 프롬프트 설계 원리 요약

| 원리 | 적용 방식 |
|---|---|
| **역할 부여(Role)** | "당신은 ~ 카피라이터/블로거" 로 톤·전문성 고정 |
| **포맷 제약(Format)** | 글자 수(280자), 해시태그 개수(10개), 소제목(##) 명시 |
| **톤앤매너 변수화** | `{톤}` 을 외부 입력으로 빼서 A/B(친근한 vs 전문적인) 전환 |
| **출력 정제** | "본문만 출력 / 다른 설명 쓰지 마세요" 로 후처리 비용 제거 |
| **언어 선택** | 텍스트는 한국어, 이미지 프롬프트는 영문(모델 안정성) |
