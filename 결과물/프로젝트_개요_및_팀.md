# 📋 프로젝트 개요 & 팀 구성

## 1. 프로젝트 개요

| 항목 | 내용 |
|---|---|
| 프로젝트명 | 소셜 콘텐츠 멀티채널 자동 생성 시스템 |
| 목표 | 하나의 주제를 입력하면 인스타그램·블로그·X(트위터)용 텍스트와 대표 이미지를 자동 생성하고 한곳에 저장 |
| 사용 도구 | n8n(자동화), Google Gemini(텍스트), Pollinations.ai(이미지) |
| 트리거/입력 | n8n Form(주제 + 톤앤매너) |
| 산출물 | 플랫폼별 콘텐츠 6세트(A/B), 대표 이미지, 프롬프트 템플릿, A/B 결론 |
| 기간 | 2026.06.23 ~ 2026.06.29 (1주) |

### 워크플로우 한눈에
```
주제 입력 폼 → Gemini 인스타그램 → Gemini 블로그 → Gemini X → 결과 조립(이미지 URL) → Notion 저장
```

## 2. 팀 구성원 역할

| 이름 | 역할 | 담당 작업 |
|---|---|---|
| 남지현 | 자동화 워크플로우 담당 | n8n 워크플로우 설계·구축(6노드), Docker 환경 구성, Gemini·Notion API 연동 |
| 최지민 | 노션 & 문서화 담당 | Notion 플랫폼별 표 3개 설계·연동, README 및 프로젝트 문서 작성, 환경설정·트러블슈팅 |
| 전종현 | 프롬프트 & 콘텐츠 담당 | 플랫폼별 프롬프트 설계(초안→수정→최종), 콘텐츠 생성·검수, A/B 테스트 |

## 3. 개인별 작업 요약

### 남지현
- n8n으로 `주제 입력 폼`부터 `Notion 저장`까지 6블록 워크플로우를 설계·구현했습니다.
- `docker-compose` + `.env.local` 키 주입 구조를 잡아 팀원 누구나 동일하게 실행할 수 있게 환경을 표준화했습니다.
- Gemini 503 자동 재시도(4회)와 플랫폼별 `database_id` 자동 분기 로직으로 안정성을 확보했습니다.

### 최지민
- 인스타·블로그·X 3개 데이터베이스를 동일 스키마(주제·톤·본문·이미지URL)로 구성하고 인티그레이션을 연결했습니다.
- 비개발자도 따라 할 수 있도록 README와 프로젝트 개요 문서를 작성했습니다.
- 환경변수 설정과 트러블슈팅(DB ID 정정, `이미지URL` 칸 타입 등)을 정리해 실행 오류를 해결했습니다.

### 전종현
- 인스타·블로그·X 각 플랫폼 특성에 맞춰 프롬프트를 초안→수정→최종으로 발전시켰습니다.
- `2026 AI 트렌드` 주제로 플랫폼별 콘텐츠 6세트를 생성하고 품질을 검수했습니다.
- 친근한/전문적인 톤 A/B 테스트를 수행하고 플랫폼별 최적 톤 결론을 도출했습니다.
