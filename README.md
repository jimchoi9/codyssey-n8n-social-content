## 0. 처음이라면? 용어부터 (3분)

### 🤖 n8n (엔에이트엔) — "자동화 조립 블록"
레고처럼 **블록(노드)을 선으로 연결해서 자동화를 만드는 도구**예요.
"주제 받기 → AI한테 글 시키기 → 이미지 만들기 → Notion에 저장하기" 를
코드 대신 **블록을 줄줄이 잇는 방식**으로 만듭니다. 그 결과물이 `workflow.json` 이에요.

### 📦 Docker (도커) — "프로그램 통째로 담은 도시락"
n8n을 내 컴퓨터에서 켜려면 보통 이것저것 깔아야 하는데, Docker는
**필요한 걸 다 담아둔 도시락**이라고 보면 돼요.
> 컴퓨터가 윈도우든 맥이든, **"도시락(Docker)만 있으면 똑같이 데워 먹을 수 있다"** → 이게 Docker의 핵심.
> 팀원들 컴퓨터가 제각각이어도 동일하게 실행돼요. `docker compose up` 한 줄이면 켜집니다.

### 🔑 "n8n 회원가입"은 진짜 가입이 아니에요
n8n을 처음 켜면 **이메일/비번을 정하라**고 하는데, n8n 회사에 가입하는 게 아니라
**"내 컴퓨터 안 n8n에 들어가는 자물쇠"** 를 만드는 거예요.
- 인증 메일 안 옴, 결제 없음, 인터넷으로 정보 안 나감 (전부 내 컴퓨터에만 저장)
- 이메일·비번 **아무거나** 넣으면 됨 (예: `me@test.com` / `test1234`)

### 🗝️ API 키 / `.env.local` — "AI 서비스 출입증"
AI한테 글을 시키려면 **그 회사(구글·노션)에 낼 출입증(API 키)** 이 필요해요.
이 출입증을 `.env.local` 이라는 **비밀 메모장**에 적어두면, n8n이 그걸 보고 호출합니다.
> ⚠️ 출입증은 **비밀번호**라서 절대 인터넷(GitHub)에 올리면 안 돼요. → `.env.local` 은 git에 안 올라가게 막아뒀습니다.

### 📊 Notion 인티그레이션 — "Notion에 글 쓸 권한"
n8n이 내 Notion 표에 저장하려면, Notion이 **"이 프로그램은 내 표에 글 써도 돼"** 라고 허락해줘야 해요.
그 허락 도장이 **인티그레이션 토큰**이고, **저장할 표마다 연결**해줘야 합니다.

## 1. 이 자동화가 하는 일

**흐름** (왼쪽 → 오른쪽으로 블록이 순서대로 실행 · 총 **6블록**):
```
[주제 입력 폼] → [Gemini 인스타] → [Gemini 블로그] → [Gemini X] → [결과 조립] → [Notion 표 저장]
```

| 블록(노드) | 하는 일 | 쓰는 도구 |
|---|---|---|
| 주제 입력 폼 | 주제 + 톤(친근한/전문적인) 입력받아 시작 | n8n Form |
| Gemini 인스타 | 짧은 캡션 + 해시태그 10개 생성 | Gemini 2.5-flash |
| Gemini 블로그 | 제목 + 소제목 + 본문(500자+) 생성 | Gemini 2.5-flash |
| Gemini X | 280자 이내 요약 트윗 생성 | Gemini 2.5-flash |
| 결과 조립 | 3개 결과 정리 + **대표 이미지 URL 생성**(무료 Pollinations) | 코드(JS) |
| Notion 표 저장 | **플랫폼별로 자기 표에** 행으로 저장 | Notion |

> 💡 이미지는 별도 블록 없이 **`결과 조립`에서 Pollinations URL을 만들어** 각 행의 커버로 넣어요. (OpenAI·결제 불필요)
> 💡 `Notion 표 저장`은 플랫폼을 보고 **database_id를 자동 선택**합니다: 인스타→인스타 표, 블로그→블로그 표, X→X 표.

> **미션 대응**: ① 입력·트리거 / ② 플랫폼별 텍스트 3종 / ③ 대표 이미지 / ④ 플랫폼별 구분 저장 / ⑤ 보너스 A/B(톤) — 전부 충족.

## 2. 준비물 (키 + Notion 표 3개)

### (1) 키 발급 → `.env.local` 에 적기

`.env.local` 을 열어 아래 값을 채웁니다. (이미지는 무료라 **OpenAI 키 불필요**)

| 적을 칸 | 발급받는 곳 | 비용 |
|---|---|---|
| `GEMINI_API_KEY` | https://aistudio.google.com/apikey | **무료** |
| `NOTION_TOKEN` | https://www.notion.so/my-integrations → "New integration" | 무료 |
| `NOTION_DB_IG` | 아래 (2)에서 만든 **인스타 표** ID | 무료 |
| `NOTION_DB_BLOG` | **블로그 표** ID | 무료 |
| `NOTION_DB_X` | **X 표** ID | 무료 |

### (2) Notion에 표(데이터베이스) **3개** 만들기

플랫폼마다 표 1개씩 — **인스타 표 / 블로그 표 / X 표**. 3개 모두 **칸 구성을 아래와 똑같이**:

| 칸 이름 | 타입 | 비고 |
|---|---|---|
| `주제` | 제목(Title) | 기본 'Name' 칸을 `주제` 로 이름만 변경 |
| `톤` | 선택(Select) | 친근한/전문적인 — 옵션 자동 생성 |
| `본문` | 텍스트(Text) | 생성된 글이 들어감 |
| `이미지URL` | URL | 대표 이미지 주소 |

> 표가 곧 플랫폼이라 `플랫폼` 칸은 필요 없어요. **A/B는 `톤` 칸으로 구분**됩니다.

각 표 우측 상단 `⋯ → 연결(Connections)` 에서 (1)의 **인티그레이션을 3개 표 모두에 연결**하세요. **(빼먹으면 저장 안 됨!)**

**DB ID 찾는 법**: 표를 풀페이지로 연 뒤 URL `notion.so/워크스페이스/`**`32자리 글자`**`?v=...` 의 가운데 32자리.

## 3. 실행하기 (순서대로 따라하기)

### STEP 1. Docker 켜기
- **Docker Desktop 앱** 실행 (고래 아이콘이 멈추면 준비 완료)

### STEP 2. n8n 켜기
이 폴더에서 터미널을 열고:
```bash
docker compose up -d
```
> ⚠️ `.env.local` 키를 **바꿨다면** 아래로 다시 켜야 반영됩니다 (켜질 때 한 번만 읽음):
> ```bash
> docker compose up -d --force-recreate
> ```

### STEP 3. n8n 들어가기
1. 브라우저 **http://localhost:5678**
2. (처음이면) 이메일·비번 **아무거나** 정하기

### STEP 4. 워크플로우 불러오기
- 우측 상단 메뉴 → **Import from File** → **`workflow.json`** 선택
- 블록 **6개**가 연결된 화면이 보이면 성공 👍 (이 화면 캡처 = "워크플로우 구조 스크린샷")

### STEP 5. 실제로 돌려보기
1. `주제 입력 폼` 블록 → **`Execute step`** → 뜨는 폼 주소 접속
2. 주제(예: `2026년 AI 트렌드`) + 톤 선택 → **제출**
3. 블록들이 **초록색**으로 흐르면 성공 → 각 블록 클릭 시 생성 결과 확인
4. **인스타 글은 인스타 표, 블로그 글은 블로그 표, X 글은 X 표**에 각각 1행씩 저장돼요

### 보너스: A/B 테스트
같은 주제로 **톤만 바꿔 2번** 실행 → 각 표에 `친근한`/`전문적인` 2행이 쌓여 비교 완성.

| 변수 | 관찰(예) | 결론(예) |
|---|---|---|
| 인스타그램 | 친근한 톤이 후킹·이모지로 반응 유도 유리 | IG는 **친근한** |
| 블로그 | 전문적인 톤이 신뢰도·SEO 유리 | 블로그는 **전문적인** |
| X | 친근한 톤이 공유 친화적 | X는 **친근한** |

## 4. 콘텐츠 퀄리티는 어느 정도?

솔직하게: **"바로 발행"보다는 1~2분 손보면 쓸 만한 수준**입니다. (자동화의 현실적 기대치)

| 항목 | 사용 도구 | 퀄리티 | 한계 |
|---|---|---|---|
| 텍스트 | **Gemini 2.5-flash** | 한국어 자연스럽고 포맷 준수 양호 | flash 등급이라 가끔 무난. **사실관계·시점은 사람 확인 필요** |
| 이미지 | **Pollinations.ai**(무료) | 소셜용으로 충분, URL 안 만료 | DALL·E보단 살짝 낮음. **이미지 속 한글은 깨짐**(그래서 텍스트 없이 생성) |

> 미션 제약("무단 자동 게시 금지")상 **사람이 검수 후 발행**하는 전제가 맞습니다.

## 5. 퀄리티 올리고 싶을 때 (어디를 고치나)

n8n에서 해당 **블록을 더블클릭** 해서 고치면 됩니다.

| 원하는 것 | 어디를 / 무엇을 |
|---|---|
| **글을 더 똑똑하게** | `Gemini 인스타/블로그/X` 블록의 주소(URL)에서 `gemini-2.5-flash` → **`gemini-2.5-pro`** (느리고 비싸지만 깊이↑) |
| **창의성 조절** | 블록 Body의 `temperature` — 낮추면(0.3) 차분, 높이면(1.0) 다채로움 |
| **브랜드 말투 입히기** | 각 Gemini 블록 프롬프트에 "우리 브랜드는 ○○한 말투. 예시…" 2~3개 추가 |
| **이미지 키우기/스타일** | `결과 조립` 블록의 Pollinations URL `width/height` 조정, 프롬프트 단어 추가 |
| **고급 이미지로** | OpenAI DALL·E 블록으로 교체(키 필요) — `.env.local`의 `OPENAI_API_KEY` 활성화 |

> 프롬프트 초안→수정→최종은 `prompts/프롬프트_템플릿.md` 에 있어요.

## 6. 다른 컴퓨터에서도 쓰기 (git)

이 폴더를 GitHub에 올리면 팀원도 받아 쓸 수 있어요. **단, 두 가지**:

1. **`.env.local`(진짜 키)은 git에 안 올라감** (보안). → 받는 사람은 `.env.local.example` 을 복사해 키를 **다시** 채움.
2. **워크플로우는 `workflow.json` 을 다시 Import** (n8n 내부 데이터는 Docker에 있어서 자동으론 안 따라옴).

**올리는 쪽**
```bash
git init
git add .
git commit -m "n8n 소셜 콘텐츠 자동화"
git remote add origin <내 GitHub 저장소 주소>
git push -u origin main
```
**받는 쪽**
```bash
git clone <저장소 주소>
cd n8n-social-content
cp .env.local.example .env.local   # ← 그리고 키 채우기
docker compose up -d               # localhost:5678 → workflow.json Import
```

## 7. 잘 안 될 때 (트러블슈팅)

| 증상 | 해결 |
|---|---|
| **Gemini 노드 503 (Service unavailable)** | 구글 서버 일시 과부하. **자동 재시도(4회)** 가 켜져 있어 대부분 통과. 계속되면 잠시 후 재실행 |
| 키를 바꿨는데 그대로임 | `.env.local` 저장 후 `docker compose up -d --force-recreate` |
| Notion 저장 실패 (400/404) | 표 칸 이름이 [2-(2)](#2-notion에-표데이터베이스-3개-만들기)와 같은지 + **3개 표 모두 인티그레이션 연결** 했는지 확인 |
| 노션에서 이미지가 안 보임 | 표의 **행을 열면** 맨 위 커버 배너로 보여요(목록에선 안 보임). URL은 `이미지URL` 칸에 있음 |
| 블로그가 중간에 잘림 | Notion 한 칸 한도(2000자)로 1900자에서 자릅니다 |
| n8n 화면이 안 열림 | Docker Desktop 켜졌는지 확인 → `docker compose up -d` → http://localhost:5678 |
| 끄고 싶을 때 | `docker compose down` (데이터 유지) |
